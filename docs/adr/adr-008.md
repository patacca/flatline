# ADR-008 Cross-Platform Order

- Status: accepted
- First needed: start of P6

## Context

After the Linux MVP, flatline needed to choose the first non-Linux host family
for supported expansion while minimizing duplicated porting work.

## Decision

Expand support macOS first and Windows second. Remove shared build and contract
blockers once, prove equivalent behavior on macOS, then take on the remaining
MSVC- and Windows-specific work.

Supported runtime-host promotion is a separate decision from wheel publication.
A host is promoted into the supported runtime-host tier only after it has
equivalent contract coverage.

Equivalent contract coverage means:

- native-enabled source and wheel install paths complete on that host
- `DecompilerSession` startup, pair enumeration, and native decompile behavior
  match the existing public contract
- the committed fixture-backed matrix and negative tests run on that host
- CI contains a dedicated continuous host lane proving the behavior
- release-facing support messaging is updated in the same change set

## Consequences

- Shared portability work is paid down before the repo tackles Windows-specific
  tail risk.
- Host support promotion requires dedicated continuous contract evidence, not
  just a successful one-off build.
- Wheel publication alone does not make a host part of the supported runtime
  contract.
- Published-wheel targets may legitimately remain outside the supported runtime
  tier until they satisfy the promotion bar above.
- This ADR defines the durable host-promotion rule; release-facing docs report
  the current support state for a given release line.
