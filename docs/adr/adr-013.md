# ADR-013 Wheel Distribution Strategy

- Status: accepted
- First needed: P6.5 matrix lock

## Context

flatline needed a durable wheel policy covering interpreter family, Python
versions, host matrix, and build tooling for public distribution.

## Decision

Ship CPython-only 64-bit wheels for `>=3.13`, currently 3.13 and 3.14, using
`cibuildwheel`. The locked matrix is manylinux x86_64, manylinux aarch64,
Windows x86_64, macOS x86_64, and macOS arm64. Linux policy remains
`manylinux_2_28` and the macOS deployment target remains `11.0`. 32-bit,
musllinux, Windows ARM64, and macOS universal2 targets stay deferred.

## Matrix Details

| Host family | Arch | Wheel tag family | Current GitHub runner |
| --- | --- | --- | --- |
| Linux | x86_64 | `manylinux_2_28_x86_64` | `ubuntu-latest` |
| Linux | aarch64 | `manylinux_2_28_aarch64` | `ubuntu-24.04-arm` |
| Windows | x86_64 | `win_amd64` | `windows-latest` |
| macOS | x86_64 | `macosx_*_x86_64` | `macos-15-intel` |
| macOS | arm64 | `macosx_*_arm64` | `macos-15` |

The matrix is intentionally 64-bit only and currently produces 10 wheels plus
1 sdist per release on the CPython 3.13/3.14 line.

## Rationale

- It matches the product direction: CPython-only, 64-bit only.
- It covers every host family flatline currently intends to distribute on.
- It uses standard GitHub-hosted native runners rather than adding a separate
  cross-build support burden.
- It keeps the Linux compatibility floor explicit via `manylinux_2_28`.
- It keeps the macOS compatibility floor explicit via deployment target
  `11.0`.

## Release Guardrails

- Wheel builds run `tools/flatline_dev/wheel_smoke.py` through `cibuildwheel`,
  so each built wheel must import, auto-discover `ghidra-sleigh`, and complete
  the wheel smoke path before publish continues.
- Release automation also smoke-tests the published artifacts back from the
  package index across the same platform and Python-version matrix.

## Deferred Targets

- 32-bit Linux and Windows wheels
- musllinux wheels
- Windows ARM64 wheels
- macOS universal2 wheels

These remain deferred for demand and maintenance reasons, not because the
public API contract requires their absence.

## Consequences

- Release automation can treat the wheel matrix as a durable distribution
  policy.
- On the current CPython 3.13/3.14 line, each release yields 10 wheels plus 1
  sdist.
- Published-wheel targets remain distinct from fully supported runtime hosts.
- Any expansion of the wheel matrix is a deliberate support decision rather than
  incidental CI growth.

## Amendment 2026-04-25

Per-wheel size cap of 80 MiB; enforced by `tools/footprint.py --max-wheel-size`.
