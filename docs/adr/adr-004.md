# ADR-004 Runtime Asset Policy

- Status: accepted
- First needed: end of P2

## Context

flatline needed a default runtime-data policy for compiled Sleigh assets while
still allowing advanced users to manage their own runtime-data trees.

## Decision

Depend on `ghidra-sleigh` for the default runtime-data experience. When
`runtime_data_dir` is omitted, flatline auto-discovers the installed companion
package's runtime-data root. The default expectation is the full multi-ISA
install. Lighter or custom setups remain explicit `runtime_data_dir`
overrides. Upstream drift in the auto-discovered companion package is surfaced
as a runtime warning rather than a silent fallback.

## Consequences

- Default installs stay self-contained for broad out-of-the-box ISA coverage.
- Users may still point to custom runtime-data roots when they need a lighter
  or locally managed setup.
- Flatline does not silently guess around runtime-data compatibility problems.
