# ADR-011 Setup Failure Taxonomy

- Status: accepted
- First needed: P3

## Context

User-fixable setup failures and unexpected flatline bugs needed distinct error
categories so callers could separate environment/configuration remediation from
internal defect handling.

## Decision

Use `configuration_error` for missing or invalid `runtime_data_dir`,
unavailable default `ghidra-sleigh` runtime data, and other user-fixable setup
failures. Reserve `internal_error` for unexpected flatline, bridge, or native
bugs.

## Consequences

- Callers can route remediation for setup failures differently from defect
  reporting.
- Error taxonomy stays aligned with the stable public contract instead of
  leaking bridge-specific failure classes.
