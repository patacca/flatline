# ADR-006 Logging and Diagnostics

- Status: accepted
- First needed: end of P2

## Context

flatline needed a durable diagnostics policy that preserved useful debugging
information without turning the library into a logging framework or leaking raw
request data.

## Decision

Emit diagnostics through startup/runtime-data `RuntimeWarning` messages plus
structured `WarningItem` and `ErrorItem` payloads. Diagnostic text may include
full filesystem paths when they help users debug their own environment. Raw
memory-image bytes are never emitted. No general-purpose public logging sink is
exposed.

## Consequences

- Users get actionable diagnostics without parsing free-form logs.
- Debuggability wins over path redaction for this library use case.
- Degraded-but-usable results can surface warnings without becoming hard
  failures.
