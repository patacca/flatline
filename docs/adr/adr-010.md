# ADR-010 Runtime Data Packaging

- Status: accepted
- First needed: start of P2

## Context

Compiled Sleigh assets and runtime-data files needed a packaging model that
kept flatline installable from PyPI without pushing that asset-build burden onto
end users.

## Decision

Package compiled runtime data in the separate `ghidra-sleigh` distribution
(`import ghidra_sleigh`). That companion package builds `sleighc` from Ghidra
C++ sources at package build time, compiles `.sla` files ahead of use, ships
them as package data, and exposes `ghidra_sleigh.get_runtime_data_dir()` for
consumers.

## Consequences

- Flatline depends on a companion runtime-data package instead of bundling those
  assets directly.
- Sleigh compilation happens ahead of runtime use rather than at install-time
  in flatline itself.
- ADR-004 defines the default flatline policy layered on top of this packaging
  mechanism.
