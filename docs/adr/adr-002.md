# ADR-002 Bridge Surface Stability

- Status: accepted
- First needed: start of P2

## Context

flatline wraps unstable upstream C++ decompiler internals. The project needed a
clear boundary between the stable public surface promised to users and the
replaceable implementation details behind it.

## Decision

Treat the public Python API as the stable compatibility boundary. The nanobind
extension module and all bridge/native C++ code are internal implementation
detail.

## Consequences

- Public compatibility commitments attach to Python request/result models,
  helpers, and error categories.
- Bridge/native code may change as needed as long as the public contract stays
  intact.
- Tests should assert contract behavior at the Python layer rather than pinning
  internal bridge structure.
