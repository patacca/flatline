# ADR-003 Determinism Oracle Level

- Status: accepted
- First needed: end of P1

## Context

The repo needed a durable way to test deterministic decompilation behavior
without overfitting to exact pretty-printing differences that are not part of
the product contract.

## Decision

Use normalized token and structure comparison as the determinism oracle instead
of exact canonical C text.

## Consequences

- Regression tests compare normalized structural output rather than raw
  formatting.
- Whitespace-only and other non-contractual presentation drift do not count as
  regressions.
- Determinism remains measurable without turning pretty-printing details into a
  compatibility promise.
